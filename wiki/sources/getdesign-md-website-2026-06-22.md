# GetDesign MD Website

## 출처와 분류

VoltAgent team이 운영하는 `getdesign.md` 홈페이지다. AI coding agent가 UI를 만들 때 참고할 `DESIGN.md` 분석을 검색·열람하는 catalog와 특정 웹사이트 분석 요청 경로를 제공한다.

- domain: ai, software, design
- content_type: website
- knowledge_role: source, tool, concept, reference
- source_quality: official
- use_for: concept, reference, inspiration

## 원문 요약

사이트는 실제 웹사이트에서 관찰한 pattern, token, rule을 `DESIGN.md` 형식으로 정리해 coding agent용 UI 생성 참고자료로 제공한다고 설명한다. 2026-06-22 snapshot에는 공개된 DESIGN.md file 75개가 표시되고, 별도의 catalog에는 요청 가능한 analysis가 300개 이상이라고 소개된다.

공개 항목은 AI·LLM platform, developer tool, backend·DevOps, productivity·SaaS, design tool, fintech, commerce, media, automotive 등으로 나뉜다. 각 항목은 해당 브랜드의 색, typography, layout, visual atmosphere를 짧게 요약한다.

## 핵심 주장

- `DESIGN.md`는 AI coding agent가 UI의 pattern, token, rule을 읽도록 만든 Markdown artifact다.
- 공개 catalog는 브랜드별 디자인 분석을 inspiration으로 제공한다.
- 특정 공개 또는 비공개 웹사이트에 대한 DESIGN.md 제작 요청도 받는다.
- 사이트는 Google Stitch의 DESIGN.md spec을 기반으로 한다고 설명한다.

## 내 프로젝트에 주는 시사점

- coding agent에게 시각 요구를 매번 prompt로 반복하기보다 repository root의 versioned [[DESIGN.md]]로 관리할 수 있다.
- [[디자인 스펙 자동화]]에서 Figma가 없는 작은 PoC는 먼저 Markdown contract로 color role, typography, spacing, component state, responsive behavior, 금지 패턴을 명시할 수 있다.
- Catalog의 브랜드 분석은 최종 디자인 복제가 아니라 vocabulary와 비교 사례를 찾는 inspiration layer로 쓰는 편이 안전하다.

## 검증 필요 주장

- `#1 collection`, `production-grade`, `real depth`, `high-quality UI`는 운영자의 마케팅 표현이며 독립 evaluation이 없다.
- 공개 file 75개와 request catalog 300개 이상은 서로 다른 범위다. 300개가 모두 즉시 사용 가능한 공개 DESIGN.md라는 뜻으로 해석하지 않는다.
- 디자인 분석은 snapshot 시점의 공개 웹 화면에서 파생되므로 원 사이트 변경, responsive state, accessibility semantics, hidden token을 완전하게 반영한다고 볼 수 없다.
- private DESIGN.md 요청의 수집 범위, 결과물 소유권, 보존 기간, 비밀 유지 조건은 실제 의뢰 전 약관으로 확인해야 한다.
