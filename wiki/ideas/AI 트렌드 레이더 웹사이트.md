# AI 트렌드 레이더 웹사이트

## 한 줄 요약

빠르게 변하는 AI·개발 트렌드를 하루 여러 번 자동 수집, 요약, 태깅해 개인이 반복 확인할 수 있는 무료 운영형 뉴스 레이더 웹사이트를 만든다.

## 문제

AI 모델, 개발 도구, 논문, 오픈소스 저장소, 기업 발표가 시간 단위로 쏟아지면서 개인이 중요한 변화를 놓치지 않고 따라가기 어렵다. 카카오톡 채팅방처럼 휘발성 높은 채널에 공유되는 정보는 빠르게 볼 수 있지만, 나중에 검색, 비교, 재사용하기 어렵다.

사용자가 원하는 것은 단순 뉴스 모음이 아니라 "내가 따라가야 할 AI 변화"를 자동으로 걸러주고, 반복적으로 볼 수 있으며, 나중에 프로젝트나 글쓰기의 자료로 재사용할 수 있는 개인용 트렌드 관측소다.

## 대상 사용자

- 1차 사용자: AI-native work, agent, LLM 개발 도구, RAG, 오픈소스 모델 흐름을 자주 확인해야 하는 나 자신.
- 2차 사용자 후보: AI 제품 기획자, 개발자, 리서처, 테크 콘텐츠 작성자, 투자/시장 관점에서 AI 인프라 변화를 보는 사람.
- 제외할 사용자: 일반 뉴스 소비자, 모든 산업 뉴스를 실시간으로 보려는 사용자, 완전한 상업 뉴스 서비스를 기대하는 사용자.

## 해결 가설

무료 또는 저비용 도구만으로도 개인용 AI 트렌드 웹사이트를 운영할 수 있다. 핵심은 처음부터 대형 검색 서비스나 실시간 알림 서비스를 만드는 것이 아니라, 소스 목록과 요약 품질을 잘 관리하는 정적 사이트 자동 발행 파이프라인을 만드는 것이다.

초기 구성은 다음처럼 잡는다.

- 수집: RSS, GitHub Trending, arXiv API, Hugging Face, OpenAI/Anthropic/Google/Meta 블로그, 주요 개발자 블로그, Hacker News, Reddit 일부, 카카오톡 공유 자료의 수동 입력.
- 처리: GitHub Actions 스케줄러가 하루 2~4회 실행되어 새 항목을 가져오고 중복을 제거한다.
- 요약: 무료 한도 또는 저비용 LLM API를 사용하되, 초기에는 로컬 규칙 기반 필터와 짧은 수동 검수도 허용한다.
- 저장: Markdown 또는 JSON으로 원본 링크, 제목, 요약, 태그, 중요도, 수집 시각을 저장한다.
- 발행: GitHub Pages, Cloudflare Pages, 또는 Vercel 무료 플랜으로 정적 웹사이트를 배포한다.
- 장기 지식화: 중요한 항목만 [[LLM Wiki]] source 후보로 승격한다.

## 기존 대안

### Trend Chaser

사용자 제공 설명에 따르면 Trend Chaser는 Claude, GitHub Actions, Obsidian 기반으로 하루 4회 자동 큐레이션되는 AI/개발 뉴스 브리프 사이트다. Anthropic, OpenAI, Hugging Face, arXiv, GitHub Trending 등 150개 이상의 소스를 추적하고 시간 단위 트렌드를 게시하는 방향으로 보인다.

확인 메모: `https://www.taewoopark.com/trendchaser`는 공개 HTML에서 본문 텍스트가 거의 노출되지 않아, 현재 검토에서는 사용자가 제공한 설명을 주요 근거로 삼는다. 이후 실제 UI, 발행 형식, 데이터 구조, 검색 가능성, 아카이브 구조는 브라우저에서 직접 확인해야 한다.

### byul.ai

`https://byul.ai/`는 경제 뉴스 중심의 실시간 알림과 요약 서비스다. 공개 페이지 기준으로 실시간 경제 뉴스, watchlist, 차트 시점과 뉴스 연결, 유사 과거 패턴, AI 리서치, 무료/유료 플랜을 제시한다.

내 아이디어에 주는 시사점은 다음과 같다.

- 단순 뉴스 피드보다 "내 관심 목록과 연결된 뉴스"가 더 강한 사용 가치를 만든다.
- 시간순 타임라인, 중요도 필터, 알림, 리서치 질의는 개인용 트렌드 레이더에도 유용하다.
- 다만 byul.ai는 경제·투자 뉴스에 최적화되어 있으므로, AI/개발 트렌드 사이트는 논문, 모델, 저장소, 제품 릴리스, 개발자 커뮤니티 신호를 별도 데이터 타입으로 다뤄야 한다.

## 빠른 검증 방법

1주 안에 완성 가능한 검증 범위는 "개인용 정적 피드"다.

- 소스 20개만 선정한다: 공식 블로그 5개, GitHub/Hugging Face/arXiv 5개, 커뮤니티/뉴스레터 10개.
- 하루 2회 GitHub Actions로 수집한다.
- 각 항목을 `중요도`, `분야`, `왜 봐야 하는가`, `원문 링크` 4개 필드로 저장한다.
- 웹사이트 첫 화면에는 "오늘의 핵심 10개", "최근 24시간", "모델/에이전트/도구/논문/시장" 필터만 제공한다.
- 7일 동안 내가 매일 들어가서 실제로 읽는지, 기존 카카오톡/뉴스레터 확인 시간을 줄이는지 기록한다.

검증 기준은 다음이다.

- 하루 5분 안에 "오늘 놓치면 안 되는 AI 변화"를 파악할 수 있다.
- 중복이나 낮은 품질 항목이 전체의 30%를 넘지 않는다.
- 최소 3개 이상의 항목이 위키 ingest, 글감, 프로젝트 판단에 재사용된다.
- 수동 개입 시간이 하루 10분 이하로 유지된다.

## 예상 난이도

- 기술 난이도: 중간. RSS/API 수집과 정적 사이트 생성은 낮지만, 중복 제거, 중요도 평가, 태그 품질 관리가 어렵다.
- 운영 난이도: 높음. 소스가 깨지고, API 제한이 생기고, 요약 품질이 흔들릴 수 있다.
- 비용 리스크: 낮음에서 중간. 수집과 배포는 무료로 가능하지만, LLM 요약을 많이 돌리면 비용이 생긴다.
- 법적/윤리적 리스크: 중간. 원문 전문 재게시를 피하고, 짧은 요약과 링크 중심으로 운영해야 한다.

## 재사용 가능성

- [[LLM Wiki]]의 source 후보 발굴 파이프라인으로 재사용할 수 있다.
- [[Agentic Workflow]] 실험용으로 수집, 필터링, 요약, 발행, 검수 단계를 분리해 테스트할 수 있다.
- AI 뉴스레터, 블로그 글감, 발표 자료 준비, 프로젝트 아이디어 선별에 재사용할 수 있다.
- 장기적으로는 개인용 "AI radar"뿐 아니라 금융, 개발 도구, 스타트업, 정책 등 다른 도메인으로 복제할 수 있다.

## 무료 운영 계획

초기에는 완전 무료 운영을 목표로 한다.

- 저장소: GitHub public 또는 private repository.
- 자동 실행: GitHub Actions scheduled workflow. 하루 2회로 시작하고 안정화 후 4회로 늘린다.
- 데이터 저장: `data/items/YYYY-MM-DD.json` 또는 Markdown 파일. DB 없이 시작한다.
- 웹 프레임워크: Astro, Next.js static export, 또는 단순 Eleventy. 검색이 중요하면 Pagefind를 붙인다.
- 배포: GitHub Pages 또는 Cloudflare Pages. 서버리스 함수 없이 정적 사이트로 시작한다.
- 요약 비용 절감: 처음에는 제목/본문 일부 기반 규칙 필터, GitHub repository metadata, RSS summary를 우선 사용한다. LLM 요약은 중요도 높은 항목에만 적용한다.
- 검색: Pagefind 같은 정적 검색으로 시작한다.
- 태그: 초기에는 규칙 기반 키워드 태그를 쓰고, 나중에 LLM 태그 보정으로 확장한다.
- 그래프맵: 첫 버전에서는 제외한다. 항목-태그-소스 관계가 2주 이상 쌓인 뒤 Obsidian graph 또는 웹 graph 시각화를 검토한다.

## 승격 조건

- 7일 PoC에서 내가 매일 1회 이상 사용한다.
- 수집 항목 중 저장 가치가 있는 항목을 하루 3개 이상 발견한다.
- 무료 배포와 자동 실행이 안정적으로 돌아간다.
- 카카오톡 공유 자료를 수동 입력해도 아카이브 가치가 생긴다.
- 검색과 태그가 없어도 기본 타임라인만으로 재방문 가치가 있다.

## 보류 조건

- 매일 확인할 만큼 정보 품질이 높지 않다.
- 수집 소스 유지보수 시간이 뉴스 읽는 시간보다 커진다.
- LLM 요약 비용이 개인용 도구 가치보다 빠르게 커진다.
- 저작권 리스크가 있는 원문 재게시 구조가 필요해진다.
- 기존 서비스와 비교해 "내 관심사에 맞춘 필터"라는 차별점이 생기지 않는다.

## 다음 행동

- 1차 소스 20개 목록을 만든다.
- 데이터 스키마를 `source`, `title`, `url`, `published_at`, `captured_at`, `summary`, `why_it_matters`, `tags`, `importance`, `dedupe_key`로 초안 작성한다.
- MVP 페이지로 승격할지 판단하기 위해 `wiki/mvps/`에 1주 실험 계획을 따로 만들 후보로 둔다.
- Trend Chaser와 byul.ai를 브라우저에서 다시 열어 UI, 피드 구조, 검색/태그/아카이브 방식, 무료 플랜/운영 구조를 추가 검토한다.
- 카카오톡 공유 링크를 수동으로 붙여 넣는 최소 입력 방식부터 정한다.
