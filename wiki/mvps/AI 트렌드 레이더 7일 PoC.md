# AI 트렌드 레이더 7일 PoC

## 목표

개인용 AI 트렌드 정적 피드가 실제 재방문 가치와 지식 재사용 가치를 만드는지 7일 안에 검증한다.

## 배경 아이디어

[[AI 트렌드 레이더 웹사이트]]는 빠르게 변하는 AI, 개발 도구, 논문, 오픈소스 흐름을 자동 수집하고 요약해 개인이 반복 확인할 수 있게 하려는 아이디어다. 이번 PoC는 제품화가 아니라 "내가 매일 볼 만한 최소 피드"가 성립하는지 확인하는 실험이다.

## 대상 사용자

- 1차 사용자: AI-native work, agent, LLM 개발 도구, RAG, 오픈소스 모델 흐름을 추적하는 나 자신.
- 2차 사용자 후보: AI 제품 기획자, 개발자, 리서처, 테크 콘텐츠 작성자.
- 제외할 사용자: 일반 뉴스 소비자, 모든 산업 뉴스를 실시간으로 보려는 사용자.

## 범위

- 1차 소스 20개를 선정한다.
- 하루 2회 수집하는 정적 피드 구조를 가정한다.
- 각 항목은 `source`, `title`, `url`, `published_at`, `captured_at`, `summary`, `why_it_matters`, `tags`, `importance`, `dedupe_key`를 가진다.
- 첫 화면은 "오늘의 핵심 10개", "최근 24시간", "모델/에이전트/도구/논문/시장" 필터만 다룬다.
- 중요한 항목만 [[LLM Wiki]] source 후보로 승격한다.

## 제외할 범위

- 실시간 알림.
- 로그인과 개인 계정.
- 유료화와 결제.
- 그래프맵.
- 완전 자동 LLM 요약.
- 원문 전문 재게시.
- 모든 도메인으로 확장하는 범용 뉴스 서비스.

## 성공 기준

- 7일 중 5일 이상 직접 사용한다.
- 하루 5분 안에 "오늘 놓치면 안 되는 AI 변화"를 파악한다.
- 중복 또는 낮은 품질 항목이 전체의 30%를 넘지 않는다.
- 최소 3개 이상의 항목이 wiki ingest, 글감, 프로젝트 판단에 재사용된다.
- 수동 관리 시간이 하루 10분 이하로 유지된다.

## 1주 실험 계획

- 1일차: 소스 20개 목록을 만들고 데이터 스키마를 확정한다.
- 2일차: 샘플 항목 20개를 수동 또는 반자동으로 수집해 정적 데이터 파일로 만든다.
- 3일차: 오늘의 핵심 10개와 기본 필터가 보이는 최소 웹 화면을 만든다.
- 4일차: 중복 제거와 중요도 기준을 조정한다.
- 5일차: 실제로 5분 안에 핵심 변화를 파악할 수 있는지 기록한다.
- 6일차: wiki ingest 후보, 글감, 프로젝트 판단으로 재사용된 항목을 표시한다.
- 7일차: 계속 진행, 축소, 보류 중 하나를 결정하고 필요하면 decision record를 남긴다.

## 중단 조건

- 매일 확인할 만큼 정보 품질이 높지 않다.
- 소스 유지보수 시간이 뉴스 확인 시간을 줄이는 효과보다 커진다.
- 낮은 품질 또는 중복 항목이 전체의 30%를 넘는다.
- 기존 서비스와 비교해 "내 관심사에 맞춘 필터"라는 차별점이 생기지 않는다.
- 원문 전문 재게시 없이 가치를 만들기 어렵다.

## 필요한 자료와 도구

- 1차 소스 20개 목록.
- GitHub Actions 또는 로컬 cron 후보.
- 정적 사이트 도구 후보: Astro, Next.js static export, Eleventy 중 하나.
- 정적 검색 후보: Pagefind.
- 참고할 내부 페이지: [[AI 트렌드 레이더 웹사이트]], [[LLM Wiki]], [[Agentic Workflow]], [[Claude Code]].

- 저작권 리스크를 피하려면 짧은 요약과 링크 중심으로 운영해야 한다.
