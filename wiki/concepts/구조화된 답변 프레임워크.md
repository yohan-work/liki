# 구조화된 답변 프레임워크

## 한 줄 정의

구조화된 답변 프레임워크는 결론, 근거, 사례, 행동, 결과 같은 정보 단위를 정해진 순서로 배치해 독자가 핵심과 증거를 빠르게 이해하게 하는 글쓰기·말하기 도구다.

## 핵심 문제

답변이 배경 설명부터 길게 시작하거나 주장, 근거, 경험, 결과가 섞이면 듣는 사람이 핵심을 재구성해야 한다. 프레임워크는 내용의 질을 대신하지 않지만, 질문에 맞는 정보 순서를 제공해 전달 비용을 줄인다.

## 작동 방식

- OREO: Opinion, Reason, Example, Opinion/Offer. 주장과 설득이 중심인 글에 사용한다.
- PREP: Point, Reason, Example, Point. 의견, 지원 동기, 가치 판단처럼 결론과 근거를 묻는 질문에 사용한다.
- STAR: Situation, Task, Action, Result. 과거 행동, 문제 해결, 성과의 증거를 묻는 질문에 사용한다.
- 공통 원칙은 핵심을 먼저 제시하고, 추상적인 주장을 구체적인 근거나 행동으로 뒷받침하는 것이다.

## 좋은 사용 사례

- wiki query answer에서 결론, 근거, 예시, 다음 행동을 짧게 전달한다.
- 기술 문서에서 결정이나 문제를 먼저 밝히고 근거와 적용 사례를 설명한다.
- 면접과 발표 Q&A에서 질문 유형에 맞춰 의견과 경험을 분리한다.
- 프로젝트 회고에서 상황 설명보다 본인의 행동, 결과, 학습을 명확히 남긴다.
- agent 완료 보고에서 수행한 변경과 검증 결과를 배경보다 먼저 보여준다.

## 실패 패턴

- 모든 질문을 하나의 공식으로 답해 대화가 기계적으로 들린다.
- 결론을 반복하지만 새로운 근거나 정보가 없다.
- STAR의 Situation과 Task가 길어져 본인의 Action이 보이지 않는다.
- 팀 성과를 개인 성과로 과장하거나 검증 불가능한 숫자를 붙인다.
- 질문에 직접 답하지 않고 준비한 구조를 그대로 낭독한다.
- 프레임워크가 사실 검증, 출처, tradeoff, 불확실성 표시를 대신한다고 오해한다.

## 관련 개념과 차이

- OREO와 PREP은 모두 결론, 근거, 사례, 결론의 두괄식 구조이며 명칭과 적용 맥락이 유사하다.
- STAR는 의견을 설득하는 구조보다 과거 행동의 맥락과 증거를 재구성하는 구조다.
- [[LLM Wiki]]의 source summary는 전달 구조뿐 아니라 출처 주장, 해석, 검증 필요 주장을 분리해야 하므로 OREO나 PREP만으로 대체할 수 없다.
- [[Humanize KR]]은 논리 구조보다 번역투, 반복, 접속사, 형식명사, 시각 장식 같은 문체 패턴을 다룬다. 구조화된 답변이 기계적인 병렬과 상투적 결론으로 굳어지지 않는지 보조 점검할 수 있지만, 자동 윤문이 근거와 의미를 바꾸지 않는지 별도 검토해야 한다.

## 예시

질문이 "왜 이 방식을 선택했는가?"라면 PREP으로 결론, 이유, 사례, 결론을 제시한다. 질문이 "실제로 문제를 해결한 경험이 있는가?"라면 STAR로 상황과 역할을 짧게 밝힌 뒤 행동과 결과를 설명한다.

## 내 관점 / 임시 결론

프레임워크를 문장 생성 공식보다 정보 누락 점검표로 사용하는 편이 낫다. 질문의 목적을 먼저 판단하고, 의견 질문에는 PREP/OREO, 행동 증거 질문에는 STAR를 선택하되 출처, 불확실성, 반례가 중요한 문서에서는 별도 섹션을 유지한다.
