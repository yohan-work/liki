# 에이전틱 엔진 최적화

## 정의

에이전틱 엔진 최적화(AEO, Agentic Engine Optimization)는 AI agent가 웹사이트, 문서, API, 도구 정보를 더 잘 발견하고, 이해하고, 활용할 수 있도록 공개 문서 구조와 접근 방식을 정비하는 작업이다.

## 중요한 이유

검색이 사람의 클릭 중심 인터페이스에서 answer engine과 agent 중심 인터페이스로 이동하면, 노출의 기준도 "검색 결과 몇 위인가"에서 "agent가 필요한 순간 신뢰할 수 있는 reference로 호출할 수 있는가"로 바뀔 수 있다. [[GeekNews Weekly 202616 AEO Article]]은 국내에서는 SEO가 충분히 자리 잡기도 전에 AEO, AAO, GEO 논의가 빠르게 밀려오는 상황으로 이 변화를 설명한다.

## 핵심 아이디어

- agent는 사람이 문서를 탐색하듯 오래 머무르지 않는다. 필요한 페이지를 소수 요청으로 가져와 context에 넣고 처리할 수 있으므로, 기존 analytics의 page view, scroll, click, 체류 시간은 agent 소비를 잘 포착하지 못할 수 있다.
- token length는 agent-readable documentation의 1급 운영 지표가 될 수 있다. 너무 긴 문서는 context를 압박하고, 잘림, 생략, parametric knowledge fallback, 환각을 유발할 수 있다.
- AEO stack 후보로 robots.txt, llms.txt, agent-permissions.json, MCP Server Card, API catalog, OAuth discovery, WebMCP 같은 layer가 언급된다.
- AEO audit 도구의 점수는 아직 실전 성능 지표라기보다 checklist로 보는 편이 안전하다.
- agent가 잘 읽을 수 있는 Markdown, 명확한 canonical URL, 작은 문서 단위, 권한과 capability 설명은 비용이 낮은 준비 작업으로 볼 수 있다.

## 이 위키에서의 사용

이 개념은 [[LLM Wiki]] 자체를 agent-readable knowledge surface로 관리할 때 사용한다. source summary와 concept page는 사람이 읽기 쉬운 동시에 agent가 짧게 가져와 재사용하기 쉬운 구조여야 한다.

[[OSINT]]와 연결하면 AEO는 양면적이다. 공개 discovery surface를 잘 열면 agent에게 잘 호출될 수 있지만, MCP endpoint, API catalog, permission file, repository 문서가 의도치 않은 attack surface가 될 수도 있다.
