# Lint 리포트

## 요약

첫 lint baseline이다. 전체 wiki page는 YAML frontmatter를 가지고 있고, `wiki/index.md`에는 현재 주요 concept, project, tool, comparison, question, source가 반영되어 있다. 다만 최근 템플릿 고도화 전에 생성된 source summary 일부는 영어 섹션명을 아직 포함하고 있으며, 기존 source summary에는 새 frontmatter 필드인 `source_quality`, `use_for`가 없다.

## 고립 페이지

- 뚜렷한 고립 페이지는 보이지 않는다.
- `wiki/overview.md`는 index frontmatter의 `related`에 연결되어 있으나 본문 catalog에는 직접 노출되지 않는다. 필요하면 index에 "개요" 섹션을 추가할 수 있다.

## 중복 개념

- 현재 명백한 중복 개념은 없다.
- `[[Quant Trading]]`은 SNU Newsroom과 Naver Blog 자료가 모두 연결되어 있어 향후 `[[Backtesting]]`, `[[Overfitting]]`, `[[Automated Execution]]`이 커지면 하위 개념으로 분리할 수 있다.

## 누락 링크

- `[[AI-native Work]]`가 `wiki/overview.md`에서 사용되지만 아직 페이지가 없다.
- `[[Backtesting]]`, `[[Overfitting]]`, `[[Automated Execution]]`이 source summary에서 후보 링크로 등장하지만 아직 페이지가 없다.
- source summary에서 `[[raw/articles/...]]`, `[[raw/external/...]]` 형태로 원본 경로를 Obsidian link처럼 사용하고 있다. 실제 raw file은 존재하지만, 장기적으로는 Markdown path link 또는 `original_path` frontmatter만 사용하는 방식으로 정리할지 결정이 필요하다.

## Index 누락

- `wiki/index.md`는 현재 concept, project, tool, comparison, question, source를 반영하고 있다.
- 이번 lint report는 `[[lint-2026-05-20]]`로 index에 추가했다.
- `wiki/overview.md`를 index 본문에 별도 링크로 추가할지 검토할 수 있다.

## Frontmatter 누락

- 모든 `wiki/` Markdown 파일은 YAML frontmatter로 시작한다.
- 기존 source summary 4개에는 새 기준인 `source_quality`, `use_for`가 없다. 이후 ingest부터 적용하고, 기존 source summary는 별도 정리 작업에서 보강하는 것이 좋다.

## 출처 없는 주장

- `status: seed` 또는 `evidence_level: unsourced`인 페이지가 아직 많다. 이는 초기 seed page와 운영 메모 성격의 페이지에는 허용된다.
- `[[RAG vs Agentic Workflow]]`는 비교 seed page이며 아직 source-backed 결론이 아니다.
- `[[LLM Wiki 운영 흐름]]`은 프로젝트 운영 절차에 대한 내부 설명이므로 unsourced 상태가 적절하다.

## 오래된 주장

- 현재 날짜 기준으로 오래되었다고 판단할 만한 주장은 아직 없다.
- 다만 GitHub repository, public API catalog, tool 관련 자료는 시간이 지나면 빠르게 변할 수 있으므로 추후 lint에서 `updated` 날짜와 source 접근일을 함께 점검해야 한다.

## 모순

- 현재 명확한 contradiction은 발견되지 않았다.
- Quant Trading 관련 자료는 SNU Newsroom과 Naver Blog가 서로 보완 관계로 보이며, 아직 충돌하는 주장은 확인되지 않았다.
